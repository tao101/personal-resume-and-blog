**3. ParliamentConnect (parliamentconnect.com) — Solo Build**

> *A platform that automatically discovers, transcribes, and publishes parliamentary speeches across all social media platforms.*

Built this entire project solo — both frontend and backend — representing the most technically complex project of the career to date.

**Architecture & Tech Stack:**
- **Frontend:** Next.js with server actions and API routes
- **Backend:** Self-hosted Supabase for authentication, real-time features, and database
- **Video Processing Pipeline:** Python running on an **NVIDIA RTX PRO 6000**-powered server from **RunPod** (serverless GPU computing)

**Video Processing Pipeline (Custom-built):**
1. **Trigger:** Supabase database trigger fires a RunPod serverless function when a new parliament session becomes available
2. **Download:** Automatically downloads the full parliament session video
3. **Speaker Diarization:** Runs diarization on the video to generate timestamps for every spoken segment with unique speaker IDs based on voice signatures
4. **Transcription:** Transcribes each individual segment
5. **Active Speaker Detection (ASD):** Built a custom ASD process to detect the face of the active speaker in each segment
6. **Face Recognition:** Uses ML-based face recognition to match detected faces against a database of known Members of Parliament
7. **Database Update:** Updates Supabase with new, properly attributed video clips

**Results:** Achieved **98% accuracy** in speaker identification and attribution, with continuous improvement as more MP face images are added to the training set.

**Social Media Integration:**
- Self-hosted **Postiz** and integrated its API into the platform
- Users can connect their social media accounts and share parliamentary clips directly from ParliamentConnect without needing to log into the self-hosted Postiz instance

**Tech Stack:** Next.js, Supabase (self-hosted), Python, CUDA, RunPod (serverless GPU), ML face recognition, speaker diarization, Postiz (self-hosted)

### [Coolify Trigger.dev v4](https://github.com/tao101/coolify-trigger-v4)

### [Supabase Sync Script](https://github.com/tao101/supabase-sync-script) (Published on npm as `supabase-sync`)

A CLI tool to perform full one-time migrations between Supabase instances — solving the real problem of migrating from Supabase SaaS to self-hosted (or between self-hosted instances) without losing data, auth users, or storage files.

**Key Features:**
- **Full migration support:** Database schema (tables, functions, triggers, RLS policies), data (with COPY format for performance), auth users (preserves password hashes so users keep their credentials), storage (buckets and files with concurrent uploads), and database roles
- **Two modes:** Interactive (guided prompts for first-time use) and CI (automated with environment variables or config files)
- Automatic sequence reset after import to prevent primary key conflicts
- Dry run mode to preview changes without applying
- Connection testing that validates database URLs immediately
- Supports SaaS-to-self-hosted, SaaS-to-local, self-hosted-to-self-hosted, and self-hosted-to-local migrations

**Published:** [npmjs.com/package/supabase-sync](https://www.npmjs.com/package/supabase-sync)
**Tech Stack:** Node.js, PostgreSQL (pg_dump/psql), Supabase API

### [Latest Supabase Coolify Template](https://github.com/tao101/latest-supabase-coolify-template)

A Docker Compose template for deploying the latest Supabase stack on Coolify with automatic JWT/API key generation — solving the problem of Coolify's built-in Supabase template being outdated.

**Key Features:**
- Deploys the full Supabase stack: Studio, Auth (GoTrue), REST (PostgREST), Realtime, Storage, PostgreSQL, MinIO (S3-compatible), Supavisor (connection pooler), and Analytics (Logflare)
- **Automatic secure credential generation** using Coolify's magic environment variables — JWT secrets, anonymous/service role API keys, PostgreSQL passwords, and admin credentials are all auto-generated
- **Two configurations:** Development/Staging (default) and Production (optimized for 8 vCPU / 32GB RAM / NVMe servers)
- Production config includes tuned PostgreSQL settings (shared_buffers, effective_cache_size, work_mem, NVMe optimizations, slow query logging), connection pooling (Supavisor with 50 pool size, 1000 max clients), and increased file descriptors for Realtime
- Designed for **200-500 concurrent users** with **10GB+ databases** and **millions of rows**
- One-click deployment — paste Docker Compose in Coolify and deploy

**Services:** Kong (API Gateway), Studio, Auth, REST API, Realtime, Storage, PostgreSQL, MinIO, Supavisor, Analytics
